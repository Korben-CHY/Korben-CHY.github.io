android-async-http解析

Posted on 2015-11-18 | In Android

作为GitHub上Star最多的Http请求框架，我们很有必要学习下android async http的内部实现原理。

首先看官方给出的特性：

  • 进行异步HTTP请求，并在匿名回调中处理响应
  • 在UI线程之个执行Http请求
  • 使用Threadpool来限制并发
  • 使用RequestParams构建GET/POST参数
  • 支持Multipart文件上传，而且不需要第三方库
  • 只有60KB，对应用有很小的开销
  • 为参差不齐的移动联接优化了自动智能的重试机制
  • 自动解码gzip响应来支撑超高速的请求
  • 支持响应JSON解析
  • 支持本地cookie存储，把cookies保存在SharedPreferences

SmoothProgressBar 解析(CircularProgressBar)

Posted on 2015-11-12 | In Android

今天进入我们来解析第二篇开源项目：SmoothProgressBar，进一步了解CustomView。

准备

isInEditMode

在项目中用到了一个函数isInEditMode()，先来看下官方文档是怎么说的吧。

Indicates whether this View is currently in edit mode. A View is usually in edit mode when displayed within a developer tool. For instance, if this View is being drawn by a visual user interface builder, this method should return true.

到底怎样算是inEditMode呢？比如在用AndroidStudio编辑Layout时，在Priview中我们可以看到Layout的预览，这就是了。
在App实际运行时，isInEditMode()总是返回false，而在我们编辑Layout，开发工具展示给我们的预览时，isInEditMode()返回true。
因此我们可以在InEditMode()时设置一些简单的样式、颜色，这样在预览时可以看到一个基本的效果。

根据官方文档，Volley的优点如下：

  • 自动执行网络请求
  • 多并发网络连接
  • 使用标准的Http cache coherence缓存进行磁盘和内存响应
  • 支持请求优先级
  • 支持取消请求。可以取消一个单独的请求，也可以取消一个范围的请求
  • 可以灵活的定制。比如定制重试和后移功能
  • 强大的命令让你可以简单准确的使用异步的网络数据来填充UI
  • 调试和追踪工具

但是Volley并不适用于大体积的下载因为Volley所有响应的解析操作都在内存中进行。
