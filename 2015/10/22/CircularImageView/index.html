CircleImageView 解析

Posted on 2015-10-22 | In Android |

程序员与作家类似，需要不断的阅读优秀的作品，不断的实践，才能写出好的作品。
今天来看GitHub上的开源项目：CircleImageView。

View的构造方法

Custom View必须是View的子类，当然也可以是ViewGroup，TextView，Button等等了。
CircleImageView继承自ImageView。
继承View后会有三个Constructor，分别在不同的场景下使用，如果以下三个场景都需要，那么都要实现下。

自定义属性(Custom Attributes)

这里简单说明下，更详细权威的见官方的文档。这个文章也不错。
定义属性，需要以下几步。

  • 在资源文件（res/values/attrs.xml）中建立<declare-styleable>节点
  • 在<declare-styleable>节点下定义属性（Attribute）
  • 获取属性值
  • 应用属性值

我们可以看CircleImageView是怎样做的，res/values/attrs.xml。

属性由两部分组成：name和format。
name随意，当然最好遵循基本的命名规范了。format的值是固定的，常用的有以下几种：

  • reference
  • color
  • boolean
  • dimension
  • float
  • integer
  • string
  • fraction
  • enum
  • flag

当定义属性后，需要在当前布局的根节点中添加我们自己的命名空间。

然后就可以随便使用了。

如果我们定义为xmlns:custom，那就是以custom打头了。

每当调用类似于setImageBitmap(Bitmap bm),setImageDrawable(Drawable drawable)等为ImageView设置图像资源的函数时，都会调用到setup()，当然，在onSizeChanged(int w, int h, int oldw, int oldh)中也应该调用setup()，然后会导致我们的ImageView重绘，就会生成CirleImageView了。

总结

总结一下吧，整个过程就是这样：

  • 使用BitmapShader，填充设置的Bitmap资源
