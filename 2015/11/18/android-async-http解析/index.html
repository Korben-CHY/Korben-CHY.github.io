android-async-http解析

Posted on 2015-11-18 | In Android |

作为GitHub上Star最多的Http请求框架，我们很有必要学习下android async http的内部实现原理。

首先看官方给出的特性：

  • 进行异步HTTP请求，并在匿名回调中处理响应
  • 在UI线程之个执行Http请求
  • 使用Threadpool来限制并发
  • 使用RequestParams构建GET/POST参数
  • 支持Multipart文件上传，而且不需要第三方库
  • 只有60KB，对应用有很小的开销
  • 为参差不齐的移动联接优化了自动智能的重试机制
  • 自动解码gzip响应来支撑超高速的请求
  • 支持响应JSON解析
  • 支持本地cookie存储，把cookies保存在SharedPreferences

执行Http请求

要执行Http请求，需要下面的步骤：

  1. 创建一个AsyncHttpClient对象
    AsyncHttpClient client = new AsyncHttpClient();
  2. 构建RequestParams参数
    RequestParams params = new RequestParams("key", "value");
  3. 调用相关的get、post方法
    其他的调用方法类似，这里不再赘述。

类图

下面这张图中展示了async http中大致的类结构关系，每一个类中并没有把所有的属性和变量写出来，只写了最基本的。

  • AsyncHttpClient中的threadPool负责分发Runnable，即AsyncHttpRequest；httpClient负责执行联网操作
  • AsyncHttpRequest继承于Runnable，它利用httpClient执行联网操作
  • RequestHandle持有AsyncHttpRequest的弱引用，负责判断请求的执行状态，并可以取消请求
  • 使用RequestParams来构建请求参数
  • ResponseHandlerInterface负责在Request执行过程中发送消息，如sendPregressMessage()、sendSuccessMessage()等

sendRequest方法做了这些事：

  • 验证参数的有效性
  • 根据请求参数创建AsyncHttpRequest对象
  • 利用threadPool，提交Runnable：AsyncHttpRequest来执行http请求

缓存RequestHandle后，就可以随时取消这些请求了。

总结

到这里android async http已经大致分析完了，简单来说，就是

  • AsyncHttpRequest，即Runnable，创建请求，包括所有的请求参数
  • ThreadPool提交Runnable执行请求
  • HttpClient实际执行请求
  • Handler分发消息，跟踪处理过程，处理请求结果
