SmoothProgressBar 解析(CircularProgressBar)

Posted on 2015-11-12   |   In Android

今天进入我们来解析第二篇开源项目：SmoothProgressBar，进一步了解CustomView。

准备

isInEditMode

在项目中用到了一个函数isInEditMode()，先来看下官方文档是怎么说的吧。

Indicates whether this View is currently in edit mode. A View is usually in edit mode when displayed within a developer tool. For instance, if this View is being drawn by a visual user interface builder, this method should return true.

到底怎样算是inEditMode呢？比如在用AndroidStudio编辑Layout时，在Priview中我们可以看到Layout的预览，这就是了。
在App实际运行时，isInEditMode()总是返回false，而在我们编辑Layout，开发工具展示给我们的预览时，isInEditMode()返回true。
因此我们可以在InEditMode()时设置一些简单的样式、颜色，这样在预览时可以看到一个基本的效果。

ValueAnimator

我们还要准备下ValueAnimator的知识。
先看ValueAnimator的官方说明：

This class provides a simple timing engine for running animations which calculate animated values and set them on target objects.
There is a single timing pulse that all animations use. It runs in a custom handler to ensure that property changes happen on the UI thread.
By default, ValueAnimator uses non-linear time interpolation, via the AccelerateDecelerateInterpolator class, which accelerates into and decelerates out of an animation. This behavior can be changed by calling setInterpolator(TimeInterpolator)).

用我拙劣的英语为大家大致译下：

这个类为运行的动画提供了一个时间引擎，它计算动画的值并把值设定到目标对象上。
所有的动画使用单一的定时脉冲，这个脉冲运行在自定义的Handler中以确保属性的改变发生在UI线程。
默认情况下，ValueAnimator使用非线性的插值器，从而实现动画的加速和减速。可以通过函数setInterpolator(TimeInterpolator)设置自己的插值器。

Drawable的四个抽象方法

getOpacity()的返回值是什么意思呢？
PixelFormat有四个值：

  • TRANSLUCENT：直译就是半透明，下层图案部分可见；
  • TRANSPARENT：完全透明，下层图案完全可见；
  • OPAQUE：完全不透明，会完全覆盖下层的图案，使下层的图案不可见；
  • UNKNOWN：未知。

由于ProgressBar除边框会把下层图案覆盖，使下层图案不可见外，中心区域并不会覆盖下层图案，因此这里选用了这个属性。

总结

  • 函数draw负责画弧
  • mRotationAnimator不断改变弧线的起始位置
  • mSweepAppearingAnimator不断增加sweepAngle的值，使弧线不断增长
  • mSweepDisappearingAnimator不断减小sweepAngle的值，并通过转换，使弧线末端保持不变，使弧线不断减小
  • mEndAnimator负责使弧线消失
